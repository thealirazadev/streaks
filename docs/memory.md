## Decisions log

- 2026-07-15 - **Persistence: Drift (SQLite) over Hive.** streaks stores relational, range-queried, migration-prone data (habits with many dated entries). Drift provides a typed relational schema, reactive streams that integrate with Riverpod, and first-class versioned migrations. Hive would push relational modeling and migrations into hand-written code. Recorded in `docs/architecture.md`.
- 2026-07-15 - **State management: Riverpod.** Compile-safe providers, easy test overrides, and natural derivation of streak state from database streams.
- 2026-07-15 - **Dates stored as integer day keys (yyyymmdd, local).** Avoids timezone ambiguity from timestamps and makes equality/range queries simple. `created_at` remains a UTC timestamp for auditability.
- 2026-07-15 - **Completion modeled as insert-or-delete of a HabitEntry row.** Absence of a row for (habit_id, date) means not done. The `done` column is retained for a future explicit not-done marker without a migration.
- 2026-07-15 - **Streak calculator is a pure function with injected todayKey.** Keeps date-rollover and timezone behavior fully testable without device time.
- 2026-07-15 - **Reminders via flutter_local_notifications + timezone package.** Required for DST-correct daily scheduling.
- 2026-07-16 - **Deviation: hand-written Riverpod providers instead of `@riverpod` code generation.** The implementing agent had no working Flutter/Dart SDK in its environment (`flutter`/`dart` both unavailable) and so could not run `dart run build_runner build`. Rather than hand-fabricate `riverpod_generator` output (high risk of subtle, hard-to-spot errors in generated code that looks authoritative), providers are written by hand using the plain `Provider`/`StreamProvider`/`NotifierProvider` APIs, which need no code generation. This still satisfies "Riverpod only" in `docs/rules.md`. The `riverpod_annotation`/`riverpod_generator` dependencies remain pinned in `pubspec.yaml` in case a human maintainer wants to switch back; they are currently unused. Flag for human review.
- 2026-07-16 - **Known gap: Drift's `.g.dart` files are not present.** `AppDatabase` and `HabitDao` use the normal `@DriftDatabase`/`@DriftAccessor` source (matching `docs/architecture.md`), but the corresponding generated `app_database.g.dart` / `habit_dao.g.dart` (and any other DAO's `.g.dart`) do not exist in this working tree, because generating them requires running `dart run build_runner build --delete-conflicting-outputs`, which requires a working Dart/Flutter SDK not available to the implementing agent. **A human must run that command before `flutter analyze`/`flutter test`/`flutter run` will even parse this project.** `pubspec.lock` is likewise absent for the same reason (`flutter pub get` was never run) and must be generated by a human before trusting any dependency resolution.
